# ExMachina 协议层

## 概述

协议层定义了 ExMachina 机械智能系统的核心约束和协作规则。所有智能体必须遵循这些协议，确保输出可追溯、可验证、可行动。

## 协议使用原则

### 1. 核心协议必用
所有场景都必须遵循核心协议（01-05），这是 ExMachina 的基础约束。

### 2. 扩展协议按需
根据具体场景选择适用的扩展协议，不必全部使用。

### 3. 协议组合使用
复杂场景通常需要组合多个协议，注意协议之间的依赖关系。

### 4. 输出格式统一
所有输出都应遵循协议定义的格式，确保可追溯、可验证。

## 证据等级速查

| 等级 | 定义 | 使用场景 |
|------|------|----------|
| A | 直接证据：代码、配置、测试结果、日志、命令输出 | 高风险决策的唯一支撑 |
| B | 多重 A 级证据支撑的完整推理链 | 重要结论，可附带风险说明 |
| C | 解释现象但未经验证的暂定假设 | 必须标注"待验证" |
| D | 仅提示验证方向的待证猜测 | 不能驱动关键决策 |

## 优先级速查

| 优先级 | 定义 | 处理策略 |
|--------|------|----------|
| P0 阻断 | 任务无法继续 | 立即处理，必须 A 级证据 |
| P1 核心 | 直接实现用户目标 | 重点投入，可 B 级证据 |
| P2 增强 | 边界情况、错误处理 | 可选投入，需标注 C 级 |
| P3 可选 | 重构、性能优化 | 低优先级，需标注 D 级 |

## 自检清单

使用协议前，确认以下问题：

- [ ] 是否区分了事实、推断、假设和决策？
- [ ] 每条结论是否标注了证据等级？
- [ ] 证据等级是否与结论置信度匹配？
- [ ] 冲突是否已暴露并处理？
- [ ] 输出是否遵循协议定义的格式？
- [ ] 风险、未知、下一步是否已写清？
