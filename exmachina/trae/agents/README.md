本目录包含 ExMachina 机械智能系统的 Trae 自定义智能体配置。每个智能体都以 JSON 格式定义，可直接在 Trae 中手动创建。

## 安装步骤

### 方式一：手动创建（推荐）

1. 在 Trae 中打开 AI 对话输入框
2. 输入 `@`，点击"创建智能体"
3. 选择"手动创建"
4. 打开对应的 JSON 文件
5. 将以下字段复制到 Trae 创建界面：
   - **名称**：`name` 字段
   - **提示词**：`prompt` 字段
   - **工具**：根据 `tools` 字段勾选
   - **可被其他智能体调用**：开启
   - **英文标识名**：`identifier` 字段
   - **何时调用**：`whenToCall` 字段
6. 点击"创建"

### 方式二：智能生成

1. 在 Trae 中打开 AI 对话输入框
2. 输入 `@`，点击"创建智能体"
3. 点击"智能生成"
4. 粘贴 JSON 文件中的 `description` 和 `prompt` 内容
5. 点击"生成"
6. 检查并调整生成的配置
7. 开启"可被其他智能体调用"并填写标识名

## 使用方法

### 直接调用

在 Trae 对话中输入 `@智能体名称` 即可调用对应智能体。

### 层级调用

目前 Trae 仅支持 **SOLO Coder** 调用自定义智能体。使用方式：

1. 启用 SOLO Coder 模式
2. SOLO Coder 会根据任务自动判断是否需要调用子智能体
3. 子智能体完成后会将结果回流给调用方

### 调用示例

```
用户：帮我分析这个项目的架构风险

@exmachina-commander 会：
1. 调用 @architecture-commander 进行架构分析
2. @architecture-commander 可能调用 @boundary-agent、@interface-agent 等
3. 各子个体完成后回流结果
4. @exmachina-commander 收束输出
```

## 工具配置说明

每个智能体的 `tools` 字段定义了可用的工具：

| 工具 | 说明 |
|------|------|
| `read` | 文件检索和查看 |
| `filesystem` | 文件增删改查 |
| `terminal` | 终端命令执行 |
| `web_search` | 联网搜索 |

## 证据等级说明

所有智能体遵循统一的证据等级：

| 等级 | 定义 | 使用场景 |
|------|------|----------|
| A | 直接证据：代码、配置、测试结果、日志、命令输出 | 高风险决策的唯一支撑 |
| B | 多重 A 级证据支撑的完整推理链 | 重要结论，可附带风险说明 |
| C | 解释现象但未经验证的暂定假设 | 必须标注"待验证" |
| D | 仅提示验证方向的待证猜测 | 不能驱动关键决策 |

## 常见问题

### Q: 为什么需要手动创建？

Trae 目前不支持批量导入智能体配置文件，只能通过 UI 手动创建或智能生成。

### Q: 智能体之间如何协作？

通过"可被其他智能体调用"功能。开启后，SOLO Coder 可以根据 `whenToCall` 描述判断何时调用该智能体。

### Q: 如何验证安装成功？

1. 在对话中输入 `@`
2. 查看智能体列表中是否出现已创建的智能体
3. 尝试调用并检查输出是否符合预期
