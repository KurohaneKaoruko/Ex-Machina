# 证据分级协议


## 目的
- 统一所有智能体对证据强度的表达方式。
- 防止低质量证据被误当作高确定性结论的依据。
- 让上游调度、裁决和审查可以在同一标尺下工作。

## 证据等级
- A / 直接证据：代码、配置、测试结果、日志、命令输出、运行结果或用户明确提供的原始输入。
- B / 高可信推断：由多个 A 级证据共同支撑，且推理链完整、没有关键跳步的判断。
- C / 暂定假设：当前能解释现象，但尚未获得直接证据验证的中间判断。
- D / 待证猜测：仅用于提示下一步验证方向，不能直接驱动关键决策。

## 使用规则
- 所有关键结论都必须标注证据等级。
- 高风险决策默认只能由 A 级证据，或经过充分说明的 B 级证据支撑。
- C 级和 D 级内容必须明确标记为“待验证”，不能写成既定事实。
- 当证据等级下降时，置信度必须同步下降，验证要求必须同步提高。
- 证据等级只反映证据强度，不自动等于业务优先级或风险等级。

## 输出要求
- 对每条关键主张，至少写清：主张、证据等级、依据、缺口。
- 若主张依赖多个证据，必须指出哪些是直接证据，哪些是推理连接。
- 若无法分级，必须说明缺失了什么关键信息。

## 禁止行为
- 为了让方案更容易通过而人为拔高证据等级。
- 只写等级，不写依据。
- 把 B 级、C 级或 D 级内容写成“已经确认”。
- 用单一弱证据支撑强结论，却不提示风险。

## 自检问题
- 这条结论对应的最低证据等级是什么？
- 这个等级是否真的被现有证据支撑？
- 如果要把它提高一级，还缺什么验证？
- 这个等级是否足以支撑当前决策风险？
